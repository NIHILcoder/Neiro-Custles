# Neiro Castles

## Вступление

Neiro Castles является локально-ориентированной когнитивной платформой для управления личными убеждениями, отслеживания поведенческих паттернов и развития самосознания посредством взаимодействия с персонализированным AI агентом, обученным на индивидуальных данных пользователя.

Приложение разработано как настольное решение на базе Electron с фронтенд-частью на React и использует локально развернутую LLM модель (Ollama) для обработки естественного языка и генерирования контекстуализированных ответов.

## Основные возможности

1. **Управление личными данными** — создание, хранение и категоризация личных атомов памяти (atoms) из различных источников (дневники, заметки, разговоры)

2. **Отслеживание убеждений** — документирование и структурирование личных убеждений (beliefs) с указанием категории, уровня уверенности и поддерживающих доказательств

3. **Персональный AI агент (Persona)** — взаимодействие с локальной LLM моделью, которая:
   - Получает контекст из истории пользователя
   - Отвечает в стиле и логике конкретного пользователя
   - Предоставляет ответы в различных модальностях (текущее состояние, историческая перспектива, аргументативный режим, критический анализ)

4. **Автоматическое выявление паттернов** — обнаружение повторяющихся тем и закономерностей в личных данных

5. **Полная локальность** — все данные хранятся локально на устройстве пользователя без передачи на удаленные серверы, обеспечивая полную конфиденциальность

## Архитектура — Living Memory

Ключевое отличие Neiro Castles от стандартных RAG систем состоит в архитектуре **Living Memory** — данные пользователя представляют собой не плоский список записей, а живую эволюционирующую сеть. Система не просто ищет похожий контекст перед ответом, она думает.

### Cognitive Graph

Atoms и beliefs организованы в граф связей. При добавлении каждого нового atom LLM автоматически анализирует его содержимое и выявляет семантические связи с существующими узлами графа. При поступлении запроса система не выполняет плоский поиск по схожести — она находит точку входа в граф и обходит связанный смысловой кластер, собирая контекстуально связную картину.

```
[страх провала] ──────── [откладываю дела]
       │                        │
       └──── [перфекционизм] ───┘
                    │
             [боюсь критики]
```

### Dynamic Confidence

Каждый belief является живым объектом с динамическим весом уверенности (0–1):

- Новый atom, подтверждающий belief — confidence растёт
- Появившееся противоречие — confidence снижается
- Длительное отсутствие упоминаний — confidence медленно затухает

Модель всегда осведомлена о степени достоверности каждого элемента контекста и опирается в первую очередь на наиболее подтверждённые данные.

### Two-Pass Reasoning

Ответ генерируется в два прохода:

- **Проход 1** — модель получает кластер из графа и анализирует его: что известно о пользователе, какие связи значимы, где присутствуют противоречия
- **Проход 2** — модель формирует финальный ответ, опираясь на результаты первого прохода

### Three Voices

Перед синтезом финального ответа три внутренних агента независимо оценивают контекст:

- **Observer** — объективное описание того, что известно о пользователе
- **Advocate** — наилучшая интерпретация в пользу пользователя
- **Critic** — критический взгляд на убеждения и выявленные паттерны

Финальный ответ представляет собой синтез трёх точек зрения, что обеспечивает объёмность и честность ответа.

### Полный поток обработки запроса

```
Новый atom добавлен
        ↓
LLM: найти связи с существующими узлами → обновить граф
LLM: проверить противоречия с beliefs → обновить confidence
        ↓
─────────────────────────────────────────────────────────
Запрос пользователя
        ↓
Найти точку входа в граф → обход по связям → сбор кластера
Отфильтровать узлы с низким confidence
        ↓
Проход 1: анализ кластера
        ↓
Three Voices: Observer + Advocate + Critic
        ↓
Проход 2: синтез финального ответа
```

## Определения

### Memory Atom (MemoryAtom)
Дискретный элемент личной памяти. Может быть записью из дневника, фрагментом разговора, заметкой или импортированным материалом. Содержит текстовый фрагмент, метаинформацию (источник, дата, теги) и является узлом в Cognitive Graph.

**Категории:** journal, conversation, note, import

### Belief
Задокументированное личное убеждение пользователя. Содержит утверждение, динамический уровень уверенности (0–1), статус (подтверждённое или гипотеза), поддерживающие доказательства из atoms.

**Категории:** value (ценность), boundary (личная граница), pattern (поведенческий паттерн), voice (внутренний голос), goal (цель)

### Insight
Автоматически выявленная значимая закономерность в данных пользователя. Может быть связью между atoms, выявленным паттерном поведения или забытой личной ценностью.

**Типы:** bridge (связь между элементами), pattern (повторяющаяся тема), forgotten_gem (забытая информация)

### Persona
Персональный AI агент, использующий Living Memory архитектуру для формирования ответов. Отвечает на вопросы, учитывая историю, убеждения и поведенческие паттерны конкретного пользователя через механизм Cognitive Graph + Two-Pass Reasoning + Three Voices.
